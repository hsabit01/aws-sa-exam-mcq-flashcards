## EC2 Basics

### What is it?
Amazon EC2 stands for Elastic Compute Cloud.

It gives you virtual servers in AWS.

You use EC2 when you want full control of the operating system, storage, networking, and installed software.

### How it works?
You launch an EC2 instance from an Amazon Machine Image (AMI).

You choose the instance type, storage, network, and security settings.

AWS runs that virtual machine for you in an Availability Zone inside a Region.

You connect to it, install your app, and manage it like a normal server.

### Use Case
A company wants to host a web application on Linux and install custom software.

They launch EC2 instances, attach EBS volumes, and place them behind an Application Load Balancer.

### Exam Tip
Think of EC2 when the question needs a virtual server with OS-level control.

Common clues are: “install software,” “lift and shift,” “custom application,” “admin access,” or “full control of the server.”

Trap: if the question wants serverless or no server management, EC2 is usually not the best answer.

## EC2 Sizing & Configuration Options

### What is it?
This means choosing the right EC2 setup for your workload.

You choose CPU, memory, storage, network, operating system, and other options.

### How it works?
When launching EC2, you pick an instance type such as general purpose or memory optimized.

You also choose the AMI, EBS volume size and type, security group, key pair, VPC, subnet, and IAM role.

You can also enable Auto Scaling, load balancing, detailed monitoring, and user data scripts.

### Use Case
A small web app may need a general purpose instance with moderate CPU and memory.

A database server may need more memory, faster storage, and higher network performance.

### Exam Tip
Look for clues like CPU-heavy, memory-heavy, burstable, high network performance, or fast SSD storage.

If the workload is unpredictable and cost matters, burstable instances may fit.

Trap: do not choose based only on low cost. The exam wants the best balance of performance, scalability, and cost.

## EC2 Instance Types

### What is it?
EC2 instance types are families of servers with different hardware profiles.

Each type is built for different workloads.

### How it works?
AWS groups instance types by purpose.

General purpose gives balanced CPU and memory.

Compute optimized gives more CPU power.

Memory optimized gives more RAM.

Storage optimized gives fast local storage.

Accelerated computing uses GPUs or special hardware.

### Use Case
A web server often uses general purpose.

A batch processing app may use compute optimized.

An in-memory cache like Redis may use memory optimized.

A machine learning or graphics workload may use accelerated computing.

### Exam Tip
Learn the pattern, not every model name.

General purpose = balanced.

Compute optimized = CPU-heavy.

Memory optimized = RAM-heavy.

Storage optimized = high IOPS or very fast local storage.

Trap: many students memorize instance names but forget workload matching. The exam mostly tests matching the workload to the family.

## Security Group

### What is it?
A security group is a virtual firewall for EC2.

It controls inbound and outbound traffic.

### How it works?
You create rules that allow specific traffic such as SSH on port 22 or HTTP on port 80.

Security groups are stateful.

That means if inbound traffic is allowed, the response traffic is automatically allowed back.

You attach the security group to EC2 instances.

### Use Case
A web server may allow HTTP from the internet and SSH only from an admin IP.

A database server may allow traffic only from the web server security group, not from the public internet.

### Exam Tip
Key clue: “instance-level firewall” usually means security group.

Security groups allow rules only. They do not create deny rules.

They are stateful.

Trap: do not confuse security groups with NACLs. NACLs are subnet-level and stateless.

## EC2 SSH Connection

### What is it?
SSH is a secure way to connect to a Linux EC2 instance from your computer.

It is mainly used for administration.

### How it works?
You launch the instance with a key pair.

The instance must allow inbound SSH on port 22 in its security group.

You connect using the private key and the instance public IP or public DNS name.

For private instances, you usually connect through a bastion host, VPN, or Session Manager.

### Use Case
An administrator wants to log in to a Linux web server to check logs or update configuration files.

They use SSH with the correct key and security group rules.

### Exam Tip
Look for clues like key pair, port 22, Linux login, terminal access, or secure remote administration.

Trap: SSH is for Linux. Windows usually uses RDP on port 3389.

Another trap: opening SSH to 0.0.0.0/0 is less secure. The exam often prefers restricted IP access or Session Manager.

## EC2 IAM Role

### What is it?
An IAM role for EC2 gives the instance temporary AWS permissions.

It lets applications on the instance access AWS services securely.

### How it works?
You create an IAM role with permissions such as reading from S3 or writing to CloudWatch Logs.

You attach the role to the EC2 instance.

The instance gets temporary credentials automatically from the Instance Metadata Service.

Your app uses those temporary credentials instead of storing access keys on the server.

### Use Case
A web app on EC2 needs to read images from S3.

You attach an IAM role that allows S3 read access.

The app can access S3 without hardcoding credentials.

### Exam Tip
A strong clue is: “EC2 needs to access another AWS service securely.”

Best answer is usually IAM role, not access keys stored in code or config files.

Trap: never choose long-term access keys on the instance when IAM role can be used.

## EC2 Purchase Options

### What is it?
EC2 purchase options are different ways to pay for EC2.

AWS gives options for flexibility, savings, or handling interruptions.

### How it works?
On-Demand is pay as you go with no long commitment.

Reserved Instances and Savings Plans reduce cost for steady usage over time.

Spot Instances use spare AWS capacity at a big discount, but AWS can interrupt them.

Dedicated Hosts or Dedicated Instances are for special compliance or licensing needs.

### Use Case
A new app with unknown traffic may start with On-Demand.

A stable production app that runs all year may use Savings Plans or Reserved Instances.

A fault-tolerant batch job may use Spot Instances to save money.

### Exam Tip
This is a very common exam topic.

On-Demand = flexible, short-term, unpredictable use.

Savings Plans or Reserved = steady, predictable workloads, lower cost.

Spot = cheapest, but can be interrupted.

Dedicated = isolation, compliance, or license requirements.

Trap: do not use Spot for critical workloads that cannot handle interruption.
